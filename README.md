# **DecodeLabs Summer Internship - Project 3**

For the third project of the DecodeLabs frontend internship, the focus shifted from CSS layouts to actual interactivity using JavaScript. To get a solid grip on how the Document Object Model (DOM) works, I built a simple, clean tally counter application.

> **[View Live Demo Here](https://rounak-56.github.io/decodelabs-project-3/)**

<br>

## Project Objective
Instead of just making things look good, the goal for this milestone was to make things *work*. The main objective was to understand the Input-Process-Output loop: listening for a user's click, using JavaScript to update the data, and dynamically changing the text on the screen without refreshing the page.

<br>

## Tech Stack & Skills Highlighted
* **JavaScript (Vanilla):** Wrote the core logic to handle state management (incrementing, decrementing, and resetting the counter).
* **DOM Manipulation:** Safely updated the HTML elements in real-time using `textContent` to reflect the current score.
* **Event Listeners:** Wired up the buttons so they actually trigger the correct JavaScript functions when a user interacts with them.
* **HTML/CSS:** Built a straightforward, clean interface. 

<br>

## Design Notes
*Since the primary focus for this project was nailing down the JavaScript logic and keeping the code completely decoupled, I kept the UI super simple. It is currently optimized for desktop and isn't fully responsive yet, so it might look a little squished if viewed on a phone!*
